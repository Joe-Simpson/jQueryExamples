jQuery

jQuery is a JavaScript library, that makes it relatively easy to implement dynamic functionality into HTML elements. The following summary is stollen striaght from codecademy.com:

"Let's use Legos as an analogy for understanding how jQuery works. With an infinite number of Legos, you could build an entire city — of course, this would take a long time. What if you were given pre-made Lego buildings, Lego roads, Lego parks, etc? You could build a city much faster. The JavaScript language represents an infinite supply of Lego blocks — the possibilities are endless but time-consuming. The pre-made Lego structures are like jQuery methods. You can use these methods to add dynamic behavior, such as .hide(), .show(), .fadeIn(), .fadeOut() etc., to HTML elements."

$()

The $ is an alias for the jQuery function. The function $() takes a parameter that targets an element and creates a jQuery object. Typically, we pass a string into $() to target elements by id, class, or tag.

.ready()

A web page must first be rendered before any dynamic behaviour can be implemented. To do so we utilise the .ready() method.

$(document).ready(() => { callbackFunction(); });

In the above example, the .ready() method is called to trigger the callbackFunction(); once the browser has finished rendering the HMTL document. The $(document) uses the special keyword document to target the HMTL document and create a jQuery object

Target by Class

To target a HMTL element by class we pass the class string to $() prepended with a ., wrapped in apostrophes.

$('.element_class')

Target by Id

To target a HMTL element by class we pass the class string to $() prepended with a #, wrapped in apostrophes.

$('#element_id')

Creating variables from jQuery objects

It is often useful to store jQuery objects as state variables.

const $itemClass = $('.item-class');

Note that the created variable name has a preceeding $. It is best convention to name all jQuery object variables as such to remind yourself and others that they are jQuery object variables.

Generic layout of an Event Listener and a Callback Function
$('.element_class').on('event_listener', () => { $('.element_class').callbackFunction(); });
Event Listener - Click

The click Event Listener triggers a Callback Function when the specified jQuery object (usually a HMTL element) registers a mouse click.

$('.element_class').on('click', () => { $('.element_class').callbackFunction(); });

In the above example, the Event Listener will trigger the callbackFucntion() method on the HMTL class element_class whenever the HMTL class is clicked.

Event Listener - mouseenter and mouseleave

The mouseenter and mouseleave Event Listeners trigger a Callback Function when a mouse cursor enters or leaves a specified jQuery object.

Generic Callback Functions - show(), hide() and toggle()

The first two do exactly what they say on the tin. show() removes the "display: none" from the style property of the HTML element (if there is one). hide() does the exact opposite. toggle() function switches the HMTL elemnt between the two different states.

Generic Callback Functions - fadeIn(), fadeOut() and fadeToggle()

They all mirror the show(), hide() and toggle() functions in functionality but apply a fade transition, the time duration of which can be set (in milliseconds) as the parameter passed to the function (fadeIn(100)).

Generic Callback Functions - slideDown(), slideUp() and slideToggle()

They all mirror the show(), hide() and toggle() functions in functionality but apply a slide transition, the time duration of which can be set (in milliseconds) as the parameter passed to the function (slideUp(100)).

Generic Callback Functions - css()

Add the css properties passed (as an object) as a parameter in the function to the specified element the Callback Function is attached to.

Generic Callback Functions - animate()

Apply the css() Callback Function over a specified period of time. Note: animate() only works on specific css properties and will not throw an error if you try to apply one that does not play ball.

Generic Callback Functions - addClass(), removeClass() and toggleClass()

Add or remove a class from the specified element. toggleClass() behaves in the same manner as previous toggles.

Chaining Events

It is possible to chain Events so you do not need to respecify the target Element each time.

$itemClass.on('click', () => {
    $itemClass.clickCallbackFunction();
}).on('mouseenter', () => {
    $itemClass.mouseEnterCallbackFunction();
}).on('mouseleave', () => {
    $itemClass.mouseLeaveCallbackFunction();
})

Where multiple elements all have the same Class, it is possible to target the specific element for which the Event Listener is triggered on.

$($itemClass).on('mouseenter', event => {
    $(event.currentTarget).addClass('photo-active');
}).on('mouseleave', event => {
    $(event.currentTarget).removeClass('photo-active');
})

In the above example, note that instead of the generic 'un-named' function '()', the special keyword event is used in its place. This property will typically be equal to the this of the function. Then, when subsequently referencing the target element for the generic Callback Function to interact with, $(event.currentTarget) is used.

.children()

You can target all the HTML elements inside an outer element. That is to say, you can target the child elements of a parent HTML element.

const $kids = $('.class').children();

The above creates a variable $kids of all children of the element with a class of "class".

.siblings()

By combining the event.currentTarget() and .siblings() it is possible to target adjacent HMTL elements.

$('.choice').on('click', event => {
    $(event.currentTarget).siblings().removeClass('selected');
});

In the above code, the .siblings() method targets elements adjacent to the clicked '.choice' and removes the class 'selected'.

.closest()

The .closest() method will traverse up the DOM tree to find a specified selector closest to it

$('.example-class-one').closest('.another-class');

In the above example, the .closest()

method is called on '.example-class' elements. The method then targets the element selected by the .closest() method with a class of '.another-class'.

.find()

Much like the .childern() method, .find() targets singular or multiple elements that are descendants of an element. Unlike .children(), it traverses all descendants of the specified element, not just the first level down.

const $items = $('.example-class).find('li');

The .find() method takes a parameter that specifies how to filter results. This parameter is just like anything you might use to select a jQuery object, ('#id', '.class', tag, etc.). .find() will return all descendants that match the passed in selector. In the example above, the .find() method will return all <li> child elements inside the '.myList' element.
